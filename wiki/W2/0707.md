## 0707: W2M5 과제 진행과 AI 활용 방식 개선

### 진행 내용
- **W2M5 과제 주제 선정:** 팀원들과 함께 직접 데이터셋을 수집하고, 이를 분석하여 비즈니스 가치를 도출할 수 있는 주제를 논의하였다.
- **카카오톡 앱에 대한 리뷰 데이터 분석 기획:** Google Play 리뷰 데이터, 식당 리뷰 데이터 등 여러 후보를 검토한 뒤 카카오톡 앱에 대한 리뷰 데이터를 분석하는 방향으로 주제를 정하였다.
- **데이터 수집 방식 검토:** Google Play 리뷰 데이터를 수집하기 위해 `google-play-scraper`와 Selenium을 비교하였고, Selenium을 활용하여 직접 수집하는 방식으로 접근해보았다.
- **데이터 분석 실습:** 수집한 리뷰 데이터를 바탕으로 wordcloud를 생성하며 텍스트 데이터에서 주요 키워드를 확인하였다.
- **복습 루틴 실행:** 아침에 데이터 모델링의 논리 데이터 모델링 관련 내용을 다시 읽고, 내가 이해한 내용을 말로 설명해보는 시간을 가졌다.

---

### 오늘 배운 내용
강의가 없는 날이었지만, 팀원들과 직접 주제를 정하고 데이터를 수집하는 과정에서 주도적으로 문제를 정의하는 일이 쉽지 않다는 것을 느꼈다. 주어진 과제를 해결하는 것과 달리, 어떤 데이터를 수집해서 어떤 비즈니스 가치를 만들지 스스로 정하는 과정에는 더 많은 고민이 필요했다.

또한 데이터 수집 방식은 단순히 라이브러리를 가져다 쓰는 것으로 끝나지 않는다는 점을 알게 되었다. Selenium으로 직접 수집을 시도하면서 스크롤 횟수, 수집 기간, 실행 중 종료 문제처럼 실제 구현 과정에서 발생하는 제약을 고려해야 했다.

---

### KPT 회고

#### **Keep - 유지할 점**
아침에 코드스쿼드에 와서 전날 정리해둔 내용을 복습하고, 데이터 모델링의 논리 데이터 모델링 관련 지식을 말로 설명해보려는 시도를 했다. 단순히 읽고 넘어가는 것이 아니라 내가 이해한 내용을 직접 설명해보니, 어떤 부분을 알고 있고 어떤 부분이 부족한지 더 잘 확인할 수 있었다.

앞으로도 작은 지식이라도 내가 배운 내용은 말로 설명할 수 있도록 연습하고 싶다. 이렇게 해야 개념을 더 깊게 이해할 수 있고, 나중에 면접이나 협업 상황에서도 내가 아는 내용을 더 명확하게 전달할 수 있을 것이다.

#### **Problem - 문제점**
W2M5 과제의 주제를 선정하는 과정에서 주도적으로 데이터를 찾고 비즈니스 가치를 만들어내는 사고가 아직 부족하다는 것을 느꼈다. 그동안은 주어진 과제를 해결하는 방식에 더 익숙했기 때문에, 직접 데이터를 수집하고 인사이트를 도출하는 방향을 잡는 데 시간이 많이 걸렸다.

또한 AI를 활용하여 코드를 작성할 때 코드의 양이 빠르게 늘어나는 문제가 있었다. 잠깐 사이에 많은 코드가 생성되다 보니, 전체 흐름을 따라가거나 각 코드가 어떤 역할을 하는지 이해하는 데 어려움이 있었다. 이 상태로 넘어가면 과제는 진행되더라도 내가 실제로 이해한 내용은 부족해질 수 있다.

#### **Try - 시도할 점**
앞으로 AI를 활용하여 코드를 작성할 때는 AI가 답변을 생성하는 동안 이전 코드나 새로 나온 코드를 바로 읽어보겠다. 코드가 완성된 뒤 한 번에 이해하려고 하기보다, 생성되는 흐름을 따라가며 함수의 역할과 데이터의 이동 과정을 확인하겠다.

코드가 길어질 경우에는 AI에게 코드 요약이나 함수별 역할 설명을 요청하여 전체 구조를 먼저 파악하겠다. 단순히 작동하는 코드를 얻는 것에서 끝내지 않고 이해하도록 노력하고 힘들겠지만 내가 직접 설명할 수 있는 수준까지 이해하는 것을 목표로 하겠다. 

데이터 수집 과정에서도 한 가지 방법에만 고정되지 않고, 실제 제약을 확인하며 수집 범위와 방식을 조정하겠다. 이번처럼 최신순으로 모든 데이터를 가져오기 어렵다면 관련도 순으로 수집하거나, 특정 기간과 수집량을 정해 현실적으로 가능한 방식으로 문제를 해결하겠다.

### 이번 주 개선 목표
- **복습 내용 말로 설명하기:** 전날 정리한 내용을 다음날 읽고, 내가 이해한 내용을 직접 말로 설명해본다.
- **AI 생성 코드 이해하기:** AI가 작성한 코드는 바로 사용하지 않고, 함수별 역할과 전체 흐름을 확인한다.
- **코드 요약 요청하기:** 코드가 길어질 때는 AI에게 요약이나 구조 설명을 요청하여 이해한 뒤 과제에 반영한다.
- **주도적으로 데이터 가치 고민하기:** 주어진 데이터만 처리하지 않고, 어떤 데이터가 어떤 비즈니스 문제를 해결할 수 있는지 먼저 생각한다.
- **현실적인 수집 방식 선택하기:** 데이터 수집 과정에서 시간, 범위, 실행 안정성을 고려하여 가능한 방식으로 조정한다.
