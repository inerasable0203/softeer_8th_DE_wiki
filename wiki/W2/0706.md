## 0706: KPT 기반 회고와 성장을 위한 피드백 루프 만들기

### 진행 내용
- **데이터 엔지니어링 수업 수강:** 개인 회고와 팀 회고의 목적, 피드백을 성장으로 연결하는 방법, 데이터 엔지니어링에서 비즈니스 맥락을 고려하는 태도에 대해 학습하였다.
- **데이터 분석 함수 실습:** `read_csv`, `sample`, `subplot`, `imshow`, `title`, `df` 등 과제에 필요한 함수들을 직접 사용하며 모르는 함수의 사용법을 확인하였다.
- **과제 수행:** 문서와 예제 코드를 참고하여 이해하지 못한 부분을 해결하고, 내 과제에 적용할 수 있도록 코드를 작성하였다.

---

### 오늘 배운 내용
지식 노동자는 계속해서 피드백을 받고 습관을 개선하는 학습 기계가 되어야 한다. 나 자신을 개선해나가지 못한다면 결국 쓸모 있는 성장을 만들 수 없기 때문에, 회고를 통해 잘못된 습관을 빠르게 발견하고 고쳐나가야 한다는 점을 배웠다.

회고는 단순히 오늘 배운 내용을 기록하는 것이 아니라, 어떤 행동이 문제로 이어졌는지 분석하고 다음 행동을 정하는 과정이다. 

또한 데이터 엔지니어링에서도 도구 사용법만 익히는 것보다 문제를 해결하는 능력과 맥락을 읽는 능력이 중요하다는 점을 알게 되었다.

---

### KPT 회고

#### **Keep - 유지할 점**
과제를 풀 때 이해하지 못하는 부분을 바로 AI를 사용하지 않고, 웹에서 문서와 실습 코드를 찾아보며 내 과제에 적용하려고 노력하였다. 그렇게 진행해도 모르는 코드가 있다면 AI의 도움을 일부 받기는 했지만, 결과만 가져오는 것이 아니라 코드가 어떤 방식으로 동작하는지 최대한 이해하려고 했다.

앞으로도 모르는 내용을 만났을 때 바로 포기하지 않고, 공식 문서나 예제 코드를 먼저 확인하고 코드를 작성하는 습관을 유지하고 싶다. AI의 도움을 받더라도 내가 설명할 수 있는 수준까지 이해하려는 태도는 계속 가져가야 한다.

#### **Problem - 문제점**
과제를 진행하면서 모르는 함수가 나왔을 때 사용법을 바로 떠올리지 못해 자주 막혔다. 특정 라이브러리 하나의 문제가 아니라, 새로운 함수나 익숙하지 않은 코드를 만났을 때 입력값, 반환값, 사용 목적을 충분히 확인하지 않고 넘어가려는 습관이 문제였다.

이 문제는 단순히 함수 이름을 모르는 것에서 끝나지 않는다. 평소에 코드를 빠르게 실행해보는 데 집중하고, 내가 사용하는 함수가 어떤 상황에서 필요한지 깊게 확인하지 않는 습관이 원인이었다. 이 습관이 반복되면 비슷한 상황을 만날 때마다 다시 검색에 의존하게 되고, 과제 수행 속도와 코드 이해도가 함께 떨어질 수 있다.

#### **Try - 시도할 점**
앞으로의 실습에서 같은 Pandas나 Matplotlib 함수를 다시 사용할 때 더 익숙하게 작성할 수 있도록, 과제에서 사용한 함수는 따로 정리해두겠다. 함수 이름만 외우는 것이 아니라 어떤 입력을 받고, 어떤 결과를 반환하며, 어떤 상황에서 사용했는지 함께 기록하겠다.

과제를 하다가 문제점이나 나중에 다시 확인해야 하는 점이 발생하면 **복습해야 할 내용**에 정리해두겠다. 하루가 끝날 때 오늘 공부하면서 다시 봐야겠다고 느낀 내용을 함께 정리하고, 다음날 학교에 와서 바로 과제를 시작하기 전에 전날 적어둔 내용을 먼저 읽어보며 놓친 개념이나 익숙하지 않은 함수를 다시 떠올리는 시간을 가지겠다.

작은 지식이라도 내가 새로 배운 내용은 말로 설명할 수 있는지 확인해보겠다. 단순히 읽고 넘어가는 것이 아니라, 내가 이해한 내용을 내 언어로 설명하는 연습을 하면 개념을 더 깊게 이해할 수 있고, 나중에 면접이나 협업 상황에서도 더 명확하게 전달할 수 있을 것이다.

또한 코드를 작성하기 전에 바로 구현부터 시작하지 않고, 먼저 슈도코드로 전체 흐름을 잡아보겠다. 이를 통해 내가 해결하려는 문제가 무엇인지, 어떤 데이터를 어떤 형태로 바꿔야 하는지 먼저 정리한 뒤 코드를 작성하는 습관을 만들고 싶다.

### 이번 주 개선 목표
- **모르는 함수 기록하기:** 과제 중 새로 알게 된 함수는 입력값, 반환값, 사용 목적을 함께 정리한다.
- **짧은 피드백 루프 만들기:** 과제를 작은 단위로 나누고, 각 단위가 끝날 때마다 내가 이해한 내용과 막힌 부분을 확인한다.
- **복습 루틴 만들기:** 과제 중 생긴 문제점이나 나중에 확인해야 하는 내용을 **복습해야 할 내용**에 정리하고, 다음날 학교에 와서 먼저 읽어본다.
- **말로 설명하는 연습하기:** 새로 배운 작은 지식이라도 내 언어로 설명할 수 있는지 확인한다.
- **문제 해결 맥락 생각하기:** 주어진 데이터만 그대로 사용하지 않고, 이 데이터를 왜 분석하는지와 더 나은 데이터가 필요한지까지 함께 고민한다.
