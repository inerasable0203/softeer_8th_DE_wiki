## 0708: 물리 데이터 모델링 학습과 AWS 배포 실습

### 진행 내용
- **데이터 모델링 수업 수강:** 물리 데이터 모델링을 중심으로 키, 정규화, 자연키와 인조키, 소프트 딜리트 등의 개념을 학습하였다.
- **AWS 실습 환경 접속:** AWS 접속 권한을 받아 ECR과 EC2에 접근하고, 로컬에서 빌드한 Docker 이미지를 ECR 레포지토리에 푸시하였다.
- **EC2 인스턴스 생성 및 설정:** EC2 인스턴스를 만들면서 보안 그룹, 역할, 스토리지, `user-data.sh`를 설정하였다.
- **자동 실행 환경 구성:** 인스턴스 생성 시 `user-data.sh`를 통해 Docker 설치, 이미지 pull, 컨테이너 실행이 자동으로 이루어지도록 구성하였다.
- **JupyterLab 실행 확인:** EC2 인스턴스가 실행된 뒤 로컬 PC에서 Docker 컨테이너 안의 코드와 JupyterLab 인터페이스에 접속하는 데 성공하였다.

---

### 오늘 배운 내용
이번 수업에서는 지난주에 배운 논리 데이터 모델링에서 더 나아가 물리 데이터 모델링을 학습하였다. 키를 어떻게 설정할지, 자연키와 인조키를 어떤 기준으로 선택할지, 소프트 딜리트를 언제 고려해야 하는지처럼 실제 데이터베이스 설계에서 마주할 수 있는 개념들을 배웠다.

AWS 실습에서는 로컬에서 만든 Docker 이미지를 ECR에 올리고, EC2 인스턴스에서 해당 이미지를 가져와 실행하는 흐름을 경험하였다. 특히 `user-data.sh`를 활용하면 인스턴스 생성과 동시에 필요한 설치와 실행 과정을 자동화할 수 있다는 점을 확인하였다.

### KPT 회고

#### **Keep - 유지할 점**
AWS 실습을 진행하면서 중간에 연결이 잘 되지 않는 문제가 있었지만, 공식 문서와 관련 자료를 차근차근 확인하며 해결하려고 했다. 문제가 생겼을 때 바로 포기하지 않고, 에러 상황을 정리하고 검색하며 따라가니 결국 JupyterLab 실행까지 성공할 수 있었다.

앞으로도 문제가 생겼을 때는 먼저 공식 문서와 신뢰할 수 있는 자료를 찾아보는 습관을 유지하고 싶다. 대부분의 문제는 이미 누군가 겪어봤을 가능성이 높기 때문에, 스스로 문제를 정리하고 해결 방법을 찾아가는 태도가 중요하다고 느꼈다.

#### **Problem - 문제점**
데이터 모델링 강의를 들으면서 지난주에 배웠던 논리 데이터 모델링 개념이 일부 헷갈렸다. 이전 내용을 충분히 복습하지 못한 상태에서 새로운 내용을 들으니, 물리 데이터 모델링을 이해하는 데 집중력이 조금 떨어졌다.

이 문제는 단순히 오늘 수업 하나를 놓친 것이 아니라, 앞에서 배운 개념이 다음 개념의 기반이 된다는 점을 제대로 관리하지 못한 결과라고 생각한다. 복습이 부족하면 새로운 내용을 배울 때 이해도가 낮아지고, 결국 나중에 다시 더 많은 시간을 써야 할 수 있다.

#### **Try - 시도할 점**
앞으로는 하루 뒤 복습과 주말 복습을 활용하여 배운 내용을 더 오래 유지할 수 있는 루틴을 만들겠다. 수업이 끝난 직후에는 핵심 개념을 간단히 정리하고, 다음날에는 전날 정리한 내용을 다시 읽으며 내가 말로 설명할 수 있는지 확인하겠다.

주말에는 한 주 동안 배운 내용을 다시 훑어보며 헷갈렸던 개념을 따로 정리하겠다. 특히 데이터 모델링처럼 앞뒤 개념이 연결되는 내용은 이전 내용을 이해하지 못하면 다음 수업의 이해도도 떨어지기 때문에, 복습해야 할 내용을 미루지 않도록 하겠다.

AWS 실습처럼 문제가 발생하는 과제에서는 에러 메시지, 시도한 방법, 해결한 방법을 간단히 기록해두겠다. 이후 비슷한 문제가 생겼을 때 같은 과정을 반복하지 않고, 내가 정리한 내용을 바탕으로 더 빠르게 해결할 수 있도록 만들고 싶다.

### 이번 주 개선 목표
- **복습 루틴 유지하기:** 과제 중 생긴 문제점이나 나중에 확인해야 하는 내용을 **복습해야 할 내용**에 정리하고, 다음날 코드스쿼드에 와서 먼저 읽어본다.
- **말로 설명하는 연습하기:** 새로 배운 작은 지식이라도 내 언어로 설명할 수 있는지 확인하고, 논리 데이터 모델링과 물리 데이터 모델링처럼 이어지는 개념은 흐름으로 설명해본다.
- **AI와 문서 활용 방식 개선하기:** AI가 작성한 코드나 공식 문서에서 찾은 해결 방법은 바로 사용하지 않고, 함수별 역할과 전체 흐름을 이해한 뒤 과제에 반영한다.
- **문제 해결 과정 기록하기:** AWS나 개발 환경에서 문제가 생기면 에러 메시지, 시도한 방법, 해결한 방법을 남겨 다음 실습에서 다시 활용할 수 있도록 한다.
- **문제 해결 맥락 생각하기:** 주어진 도구나 코드를 그대로 따라가는 것에서 끝내지 않고, 왜 이 방식이 필요한지와 더 나은 해결 방법이 있는지 함께 고민한다.
