## W2 주간 요약

이번 주는 데이터 프로젝트의 전체 흐름을 돌아보는 시간이었다.  
KPT 회고를 통해 피드백을 성장으로 연결하는 방법을 배웠다.

데이터 모델링 수업에서는 논리 모델링과 물리 모델링의 차이를 학습했다.  
AWS 실습에서는 Docker 이미지를 ECR에 올리고, EC2에서 실행하는 흐름을 경험했다.

W2M5 과제에서는 직접 데이터를 수집하고 분석 주제를 정했다.  
이 과정에서 데이터 수집보다 문제 정의와 기대 효과가 먼저라는 점을 느꼈다.

W2M5 피드백을 통해 프로젝트 대상, 전처리, 검증의 중요성을 다시 확인했다.  
AI는 결과만 받기보다 Plan - Act - Verify 흐름으로 활용해야 한다고 느꼈다.

### KPT 회고
- **Keep:** 아침에 일찍 와서 전날 배운 내용을 복습한 점은 유지하고 싶다. 단순히 다시 읽는 것에서 끝내지 않고, 내가 이해한 내용을 말로 설명해보며 부족한 부분을 확인한 점이 좋았다.
- **Problem:** 데이터 수집과 구현에 집중하다 보니 프로젝트의 전체 흐름을 놓치는 경우가 있었다. 특히 대상 정의, 기대 효과, 전처리와 검증처럼 결과물의 완성도를 높이는 과정에 대한 고민이 부족했다.
- **Try:** 다음 주에는 과제를 작은 단위로 나누고, 중간마다 방향을 점검하겠다. 프로젝트를 시작하기 전에 문제 상황과 기대 효과를 먼저 정리하고, AI나 문서를 활용할 때도 구조를 이해한 뒤 반영하겠다.

### 다음 주에 가져갈 점
- 배운 내용은 **복습해야 할 내용**에 정리한다.
- 모르는 함수와 에러 해결 과정은 따로 기록한다.
- AI가 작성한 코드는 구조를 이해한 뒤 사용한다.
- 프로젝트 시작 전에 대상과 기대 효과를 먼저 정의한다.
- 데이터 수집, 전처리, 검증, 분석 흐름을 계속 점검한다.
