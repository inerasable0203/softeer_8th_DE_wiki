## 0710: DE/DA 직무 밋업과 스쿼드 세션 회고

### 진행 내용
- **DE/DA 직무 밋업 참여:** 데이터 엔지니어와 데이터 아키텍쳐 직무에 대한 이야기를 들으며 각 직무의 역할과 필요한 역량을 생각해보았다.
- **다음 주 조원들과 스쿼드 세션 진행:** 다음 주에 함께 활동할 조 구성원들과 스쿼드 세션을 진행하며 서로의 진행 상황과 생각을 공유하였다.
- **W2M5 과제 공유:** 우리 조가 진행했던 W2M5 과제의 주제, 진행 방식, 아쉬웠던 점을 공유하며 프로젝트를 다시 돌아보았다.
- **데이터 모델링 강의 복습:** 아침에 일찍 와서 데이터 모델링 관련 강의를 다시 들으며 이번 주에 배운 내용을 복습하였다.
- **학습 태도 점검:** 점심시간 외에도 중간중간 휴대폰을 보거나 딴짓을 하는 시간이 있었다는 점을 인식하고, 다음 주 집중 방식에 대해 고민하였다.

---

### 오늘 배운 내용
오늘은 DE/DA 직무 밋업을 통해 데이터 엔지니어와 데이터 아키텍쳐가 실제로 어떤 역할을 하는지 다시 생각해볼 수 있었다. 데이터 엔지니어는 데이터를 안정적으로 수집하고 저장하며 활용 가능한 형태로 만드는 기반을 다지는 역할이 중요하고, 데이터 아키텍쳐는 데이터 구조와 흐름을 설계하여 조직에서 데이터를 일관되고 효율적으로 활용할 수 있도록 만드는 역할이 중요하다고 느꼈다.

다음 주 조 구성원들과 진행한 스쿼드 세션에서는 우리가 진행했던 W2M5 과제를 공유하였다. 과제를 설명하면서 단순히 결과를 보여주는 것보다, 어떤 문제를 해결하려고 했는지와 어떤 과정에서 아쉬움이 있었는지를 함께 전달하는 것이 중요하다는 것을 느꼈다. 특히 이전 피드백에서 나왔던 프로젝트 대상, 기대 효과, 데이터 전처리와 검증의 중요성을 다시 떠올리게 되었다.

아침에는 일찍 와서 데이터 모델링 관련 강의를 복습하였다. 한 번 들었던 내용을 다시 확인하니 논리 데이터 모델링과 물리 데이터 모델링이 연결되는 흐름을 조금 더 정리할 수 있었다. 다만 하루 중간중간 집중이 흐트러지고 휴대폰을 보게 되는 시간이 있었기 때문에, 공부 시간을 확보하는 것뿐만 아니라 그 시간의 집중도를 유지하는 것도 중요하다고 느꼈다.

---

### KPT 회고

#### **Keep - 유지할 점**
아침에 일찍 와서 데이터 모델링 관련 강의를 다시 들으며 복습한 점은 계속 유지하고 싶다. 수업이나 과제에서 한 번 접한 내용을 그대로 지나치지 않고, 다시 확인하는 시간을 가지니 이전에 헷갈렸던 개념을 조금 더 안정적으로 정리할 수 있었다.

앞으로도 하루를 시작할 때 바로 새로운 과제에 들어가기보다, 전날 배운 내용이나 이번 주에 반복해서 나오는 개념을 먼저 짧게 복습하는 루틴을 유지하겠다. 특히 데이터 모델링처럼 앞뒤 개념이 이어지는 내용은 복습을 통해 흐름으로 이해하는 것이 중요하다고 느꼈다.

#### **Problem - 문제점**
점심시간 외에도 중간중간 휴대폰을 보거나 딴짓을 하면서 집중이 끊기는 문제가 있었다. 짧은 시간이라고 생각했지만, 한 번 집중이 흐트러지면 다시 학습 흐름으로 돌아오는 데 시간이 걸렸고, 결국 실제로 공부한 시간의 밀도가 떨어졌다.

이 문제는 단순히 휴대폰을 많이 본다는 것에서 끝나지 않는다. 해야 할 일이 명확하지 않거나, 잠깐 막혔을 때 바로 다른 행동으로 빠지는 습관이 생기면 과제의 흐름을 꾸준히 이어가기 어렵다. 다음 주에는 공부 시간과 쉬는 시간을 더 분명하게 나누어야겠다고 느꼈다.

#### **Try - 시도할 점**
다음 주부터는 학습 시간에는 휴대폰을 최대한 보지 않도록 하겠다. 필요하다면 휴대폰을 눈에 잘 보이지 않는 곳에 두고, 쉬는 시간에만 확인하는 방식으로 집중 환경을 만들겠다.

과제를 진행할 때는 작은 단위로 목표를 나누고, 한 단위가 끝날 때마다 내가 이해한 내용과 막힌 부분을 짧게 확인하겠다. 이렇게 하면 중간에 집중이 흐트러지더라도 다시 돌아와야 할 지점을 찾기 쉬울 것 같다.

또한 W2M5 과제를 공유하며 느낀 아쉬움을 다음 과제에 반영하겠다. 프로젝트를 시작할 때 대상과 기대 효과를 먼저 정리하고, 데이터 수집에만 몰입하지 않도록 전처리, 검증, 분석, 개선 방향까지 전체 흐름을 계속 점검하겠다.

### 이번 주 개선 목표
- **복습 루틴과 설명 연습 유지하기:** 과제 중 생긴 문제점이나 다시 봐야 할 내용을 **복습해야 할 내용**에 정리하고, 다음날 코드스쿼드에 와서 읽어본 뒤 내 언어로 설명할 수 있는지 확인한다.
- **모르는 함수와 문제 해결 과정 기록하기:** 새로 알게 된 함수는 입력값, 반환값, 사용 목적을 함께 정리하고, 개발 환경에서 문제가 생기면 에러 메시지, 시도한 방법, 해결한 방법을 기록한다.
- **AI와 문서 활용 방식 개선하기:** AI가 작성한 코드나 공식 문서에서 찾은 해결 방법은 바로 사용하지 않고, 함수별 역할과 전체 흐름을 이해한 뒤 반영한다. 코드가 길어질 때는 요약이나 구조 설명을 요청하고, Plan - Act - Verify 흐름으로 계획, 실행, 검증 단계를 나누어 활용한다.
- **데이터와 프로젝트의 가치 먼저 고민하기:** 주어진 데이터만 처리하지 않고, 누가 어떤 상황에서 이 결과물을 필요로 하는지와 어떤 비즈니스 문제를 해결할 수 있는지 먼저 정리한다. 분석 결과가 실제 의사결정이나 개선으로 이어질 수 있는지도 함께 확인한다.
- **전체 과정 점검과 집중 환경 만들기:** 과제를 작은 단위로 나누고, 데이터 수집, 전처리, 검증, 분석, 개선 방향 중 빠진 단계가 없는지 중간마다 확인한다. 학습 시간에는 휴대폰을 멀리 두어 집중이 끊기지 않도록 하고, 피드백은 다음 과제에서 바로 적용할 행동으로 바꾼다.
