## 0703: 자동차 데이터 분석 및 SQL, 데이터 수집 파이프라인 기초 학습

### 진행 내용
- **자동차 스펙 데이터 분석 및 비즈니스 모델링:** 배기량(disp), 실린더(cyl), 무게(wt), 연비(mpg) 등 차량 데이터 간의 상관관계를 분석하고 이를 통한 비즈니스 활용 방안 도출
- **Python 연동 SQL 기초 학습:** `sqlite3` 라이브러리를 활용하여 파이썬 환경 내에서 관계형 데이터베이스 쿼리를 실행하는 방법 실습
- **다양한 SQL 문법 습득:** `ANY`, `ALL`, `IN` 등 실무에서 데이터를 다룰 때 유용하게 쓰이는 조건 문법의 이론적 학습
- **외부 API 및 공공 데이터 수집 전략 구상:** 위키피디아 API 및 IMF 엑셀 데이터를 활용한 데이터 수집 방안 기획 및 시계열 데이터 이력 관리(Versioning) 아키텍처 구상

---

### 주요 학습 내용 및 인사이트

#### 1. 데이터 분석을 통한 비즈니스 가치 창출
**핵심:** 데이터의 통계적 유의미함을 넘어, 비용 절감 및 수익 창출이라는 실질적인 경제적 가치로 연결하는 거시적인 시각이 필요하다.
- **산업별 맞춤형 활용 전략:**
  - **자동차 제조사:** 무게와 실린더 수치 조정을 통해 타겟 연비(목표 스펙)를 달성하는 R&D 최적화 지표로 활용할 수 있다.
  - **운수업체/렌터카:** 차량 생애 주기 비용을 최소화하기 위한 데이터 기반의 차량 도입(입찰) 근거로 사용할 수 있다.
  - **중고차 플랫폼:** 스펙 기반의 시세 예측 및 타겟 고객층(예: 유지비 중시 가족, 성능 선호 20대) 맞춤형 추천 알고리즘으로 발전시킬 수 있다.
- **변수 간의 상관관계 구조:** 실린더 수(cyl)와 무게(wt)가 늘어날수록 배기량(disp)도 뚜렷하게 증가(약 0.9)하는 양의 상관관계를 보인다. 반면, 이 세 가지 변수는 모두 연비(mpg)와 강한 음의 상관관계(약 -0.85 내외)를 가져, 연비 개선을 위해서는 차량의 무게와 엔진 스펙 통제가 필수적임을 확인했다.

#### 2. Python 기반 SQL 환경과 문법의 확장
**핵심:** 주어진 개발 환경의 장단점을 명확히 인지하고, 실습 가능한 범위와 이론적 지식을 분리하여 쿼리 작성 역량을 넓혀야 한다.
- **sqlite3의 한계와 유연성:** 파이썬 내부에서 가볍고 빠르게 SQL을 다룰 수 있다는 장점을 경험했다. 동시에 `sqlite3`에서 지원하지 않는 일부 문법이 존재함을 확인했으나, 이를 제약으로 두지 않고 이론적으로 학습하여 다른 RDBMS 환경에서도 활용할 수 있는 기반을 다졌다.
- **조건 검색의 정교화:** `IN`, `ANY`, `ALL` 등 평소 자주 쓰지 않던 문법을 익힘으로써, 향후 더욱 복잡하고 유연한 조건의 데이터를 필터링할 수 있는 지식을 확보했다.

#### 3. 데이터 수집 및 버전 관리(Versioning) 전략
**핵심:** 외부 데이터를 수집하여 적재할 때는 단순 저장이 아니라, 향후 시계열 분석 및 추적을 위한 데이터 정합성과 이력 관리가 필수적이다.
- **소스 다각화:** 위키피디아 API, IMF 제공 Excel 등 다양한 형태의 외부 소스에서 데이터를 수집하는 접근법을 고려했다.
- **데이터 갱신과 이력 유지:** 데이터가 갱신될 때 기존 데이터를 덮어쓰는 대신, 식별자를 달리하여 이력을 보존하는 방안을 고안했다. 이는 과거 데이터의 손실을 막고, 향후 시계열 기반의 통계 자료 조회나 데이터 변환/분석을 수행할 때 매우 중요한 설계 방식이다.
