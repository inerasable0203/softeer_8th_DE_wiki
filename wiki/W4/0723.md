## 0723: AWS Lambda 학습과 팀 과제 협업 회고

### 진행 내용
- **AWS 수업 수강:** AWS Lambda의 개념과 활용 방법을 학습하였다.
- **권한 관리의 중요성 확인:** AWS 서비스를 사용할 때 권한을 적절히 설정하고 관리해야 한다는 점을 배웠다.
- **팀 과제 진행:** 오후에는 팀원들과 과제 주제를 논의하고 프로젝트를 진행하였다.

---

### 오늘 배운 내용
오늘은 AWS 수업에서 Lambda에 대해 자세히 배웠다. 기존에는 Lambda를 잘 알지 못했지만, 이번 수업을 통해 개념과 역할을 이해할 수 있어 좋았다. 또한 AWS에서는 서비스 사용에 필요한 권한을 정확하게 설정하고 관리하는 것이 중요하다는 점을 느꼈다.

오후에는 팀 과제를 진행하였다. 팀 과제는 매번 새로운 문제를 다루기 때문에 어렵기도 하지만, 여러 아이디어를 함께 논의하는 과정에서 배울 점이 많았다.

---

### KPT 회고

#### **Keep - 유지할 점**
계속 고민해서 제안한 주제가 팀 과제의 주제로 선정된 점이 좋았다. 과제의 방향을 정하기 위해 스스로 아이디어를 생각하고 의견을 제시한 태도는 계속 유지하고 싶다.

#### **Problem - 문제점**
팀 과제를 진행하면서 협업에 충분히 기여하지 못했다. Claude에서 Superpowers 기능을 활용하는 과정이 예상보다 오래 걸렸고, 작성된 코드도 충분히 이해하지 못했다. 도구를 사용하는 데 집중하다 보니 팀원들에게 빠르게 결과를 공유하거나 코드의 내용을 설명하기가 어려웠다.

#### **Try - 시도할 점**
앞으로는 작은 기본기부터 하나씩 익히겠다. Git과 GitHub 사용법, Claude의 스킬과 플러그인 사용법을 정리하고 직접 반복해서 사용해보겠다. AI가 작성한 코드는 바로 넘기지 않고, 핵심 흐름과 각 부분의 역할을 이해한 뒤 팀원들에게 설명할 수 있도록 하겠다.

---

### 이번 주 개선 목표
- **AWS Lambda 개념 정리하기:** Lambda의 역할과 사용 흐름을 내 언어로 설명한다.
- **AWS 권한 관리 이해하기:** 서비스별로 필요한 권한과 권한 설정 시 주의할 점을 정리한다.
- **Git과 GitHub 기본기 익히기:** 팀 과제에서 필요한 명령과 협업 흐름을 직접 반복해본다.
- **Claude 도구 활용법 익히기:** 스킬과 플러그인의 역할을 확인하고 상황에 맞게 사용한다.
- **AI 생성 코드 이해하기:** 생성된 코드의 전체 흐름과 핵심 로직을 확인한 뒤 팀 작업에 반영한다.
